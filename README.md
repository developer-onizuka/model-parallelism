# モデル400GB（Attention 40GB / FFN 360GB）を、100GBのGPU 6基(model parallelism)で動かす構成の例

```
       [ 入力トークン ]
              |
      ------------------
      【 Attention層 】 <--- 全体の10%以下の重み
      ------------------
              |
      << All-Reduce >> (軽い通信)
              |
      ------------------
      【   FFN層    】 <--- 全体の90%近い重み
      ------------------
              |
      << All-to-All >> (重い重みに基づく、大量のデータ移動)
              |
```

```
【GPU 1枚(100GB)あたりのメモリ使用イメージ】

 0GB          20GB          40GB          60GB          80GB         100GB
 [-----------|---------------------------------------|-------------|-----]
      ↑                 ↑                               ↑             ↑
  Attention等        FFN / Experts                    KVキャッシュ     作業
 (TPで分割)        (EPで担当分を丸ごと保持)             (動的確保)      領域

■ 内訳の解説：
1. Attention重み (約5~10GB):
   Wq, Wk, Wv, Wo など。全層分をTPで6分割するため、1枚あたりは意外と小さい。
   
2. FFN / Experts (約60~70GB): ★ここが支配的
   MoEの場合、担当する専門家（例：32個中5〜6個）の重みを丸ごと載せるため、
   メモリの大部分をここが占有します。

3. KVキャッシュ (約20~30GB):
   推論が進むにつれて増えていく領域。ここを使い切ると「長い文章」が読めなくなる。

4. 作業領域/バッファ (約5GB):
   All-ReduceやAll-to-All通信、計算中の中間値を置くための「机」。
```
